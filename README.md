# dump_replay_and_diff
Реализация генерации трафика на основе дампа
Главная задача - анализ AVP

Описание ожидаемых действий при тестировании:

1. Сбор дампа с промышленной зоны. (Пользователь скармливает дамп вручную скрипту, который собрал на проме) 
2. Разбор и изменение дампа из п. 1 на ВМ (фильтрация входящих CCR-запросов). (Переводим через tshark в JSON-формат, на данный момент реализовано вручную, надо автоматом)
3. Проигрывание дампа на ВМ. (Класс playing.py)
4. Сбор дампа на ВМ во время проигрывания. (Реализовать как метод класса)
5. Разбор и изменение дампа из п. 4 на ВМ (фильтрация исходящих CCR-запросов). (То же, что и в пункте 2)
6. Мерж дампов из п. 2 и п. 4. (Класс comparison.py, сделано)
7. Формирование отчета о расхождениях (Класс comparison.py, сделано)

Среда исполнения - тестВМ DRA_GY. Класс playing.py будет через сокет отправлять сырые данные, принимать и декодировать.

На данный момент нам не требуется полная обработка трафика, аналогичная промышленной зоне. Главной задачей является анализ входящих и исходящих AVP-пакетов.
